# Specification Templates

Use these files in order. Keep answers short and concrete.

## Order

1. `00_DISCOVERY_QA.md` (fill this first in prose)
2. `00A_USE_CASES_AND_SCENARIOS.md` (lock concrete flows)
3. `01_SCOPE.md`
4. `02_REQUIREMENTS.md`
5. `03_ARCHITECTURE_AND_DECISIONS.md`
6. `04_OPERATIONS_AND_READINESS.md`
7. `05_BACKLOG_AND_OPEN_QUESTIONS.md`

## Writing Rules

- Prefer measurable requirements over vague statements.
- Capture decisions with rationale and alternatives.
- Mark unknowns explicitly; do not hide assumptions.
- Keep one source of truth per topic.
